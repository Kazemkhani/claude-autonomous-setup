The 9-agent hierarchy

Telegram requests hit the CEO, who plans + dispatches specialists in parallel. Each specialist has a focused tool set, a single-page system prompt, and a clear "definition of done." Nine markdown files in ~/.claude/agents/.

CEO Opus 4.7 · default entry
Strategic Orchestrator

Receives every non-trivial Telegram request. Plans, dispatches specialists in parallel, synthesizes results, replies to user. Never writes code itself — delegates everything.

Architect Opus
Designs Before Code

Used for >100 LOC features, refactors, schema changes. Outputs a plan with trade-offs, NOT code. Specialists execute the plan.

Deploy Engineer Sonnet

Vercel/Odoo.sh/Docker/EAS. Never ships to production without QA green AND security green AND explicit user approval in the current message.

QA Engineer Sonnet

Runs project test commands, writes regression tests for any bug, gates merges with a verdict report. Doesn't fix code — diagnoses.

Secret scans, OWASP review, dependency CVEs, financial-data integrity. ALWAYS called before production deploys. Returns severity verdict.

Client Handover Sonnet

How to build your own

Apple Silicon Mac, macOS 14+, ~30 minutes from zero. Replace the project names + Telegram IDs with yours.

Prerequisites

Active Claude Max subscription ($100 or $200/mo) — this is the engine. You also need brew, node ≥ 20, and Claude Code v2.1.80+. 

brew install tmux ffmpeg whisper-cpp tailscale gh
npm install -g bun @playwright/mcp playwright
npx playwright install chromium

Create a Telegram bot + add the channel plugin

On Telegram → message @BotFather → /newbot → save the token. Then in Claude Code: 

claude plugin marketplace add anthropics/claude-plugins-official
claude plugin install telegram@claude-plugins-official
claude plugin install imessage@claude-plugins-official  # backup channel

mkdir -p ~/.claude/channels/telegram
echo "TELEGRAM_BOT_TOKEN=<your-token>" > ~/.claude/channels/telegram/.env
chmod 600 ~/.claude/channels/telegram/.env

Wire the MCP servers

One command per server. Adjust args for your projects.

claude mcp add -s user mac-control -- npx -y mac-mcp-server
claude mcp add -s user desktop -- npx -y computer-use-mcp
claude mcp add -s user playwright -- /opt/homebrew/bin/playwright-mcp
claude mcp add -s user postgres -- npx -y @modelcontextprotocol/server-postgres "postgresql://localhost:5432/postgres"
claude mcp add -s user postiz \
  --transport http \
  --header "Authorization: Bearer $POSTIZ_API_KEY" \
  "https://mcp.postiz.com/mcp"
claude mcp add --transport http vercel "https://mcp.vercel.com"
claude mcp add --transport http sentry "https://mcp.sentry.dev/mcp"
claude mcp add --transport http stripe "https://mcp.stripe.com"

Patch the Telegram plugin for voice transcription

Anthropic's plugin v0.0.6 receives voice messages but doesn't transcribe. Patch server.ts at ~/.claude/plugins/cache/claude-plugins-official/telegram/0.0.6/server.ts — add a transcribeAudio() helper that pipes through ffmpeg + whisper-cli, and call it in the message:voice + message:audio handlers before handleInbound. Set the model path to ~/.claude/whisper-models/ggml-small.en.bin.

Then download the model:

mkdir -p ~/.claude/whisper-models
curl -L -o ~/.claude/whisper-models/ggml-small.en.bin \
  https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-small.en.bin
Heads up: the patch is wiped if you ever run claude plugin update telegram. Save a copy and re-apply on upgrade.

Drop in the 9 agent files

Each agent is a markdown file with YAML frontmatter — name, description (when to invoke), tools (allowed tool list), model (sonnet/opus/haiku). 

mkdir -p ~/.claude/agents

# One file per agent — see this guide's Appendix
# for the full system prompts, or copy from:
# github.com/Kazemkhani/claude-autonomous-setup

Wire the 3 hooks (defense in depth)

Even with --dangerously-skip-permissions, hooks still gate dangerous actions. Three scripts in ~/.claude/hooks/:

  • pre-bash-guard.sh — blocks force-push to main/master/production, blocks --no-verify, gates direct push to protected branches
  • post-edit-format.sh — auto-formats on every Edit/Write (prettier, ruff, gofmt, rustfmt, shfmt, swift-format) in background
  • log-prompt.sh — every UserPromptSubmit appended to ~/.claude/logs/prompts.log for audit

Add the hook entries to ~/.claude/settings.json under a top-level "hooks" object.

The launchd plist for the daemon

This is the one that keeps everything alive. Save to ~/Library/LaunchAgents/com.amir.claude-bot.plist — RunAtLoad + KeepAlive = boot-on-login, restart-on-crash. Spawns a detached tmux session running claude --channels plugin:telegram@claude-plugins-official plugin:imessage@claude-plugins-official --dangerously-skip-permissions --add-dir .... 

launchctl load ~/Library/LaunchAgents/com.amir.claude-bot.plist

Caffeinate + watchdog + briefing + backup

Four more launch agents — same pattern, each runs a tiny shell script. Caffeinate runs caffeinate -d -i -m -s as a permanent process. Watchdog checks daemon health every 30 min. Briefing runs claude -p headless Mon-Fri 9am. Config backup rsyncs the curated config to a private GitHub repo hourly with a secret-leak guard.

Tailscale + Termius for remote shell

Sign into Tailscale (free, personal use) on Mac + iPhone. Toggle Remote Login ON in System Settings → Sharing. Install Termius on iPhone. New host: Tailscale IP + your Mac username + your macOS login password. SSH in from anywhere — your Tailscale identity is the network gate.

GitHub Actions → Telegram on every CI failure

Add .github/workflows/telegram-notify.yml to each repo. Triggers on workflow_run.conclusion == failure and on push to main/master/production. Set per-repo secrets TELEGRAM_BOT_TOKEN + TELEGRAM_CHAT_ID: 

gh secret set TELEGRAM_BOT_TOKEN -R you/repo --body "$TOKEN"
gh secret set TELEGRAM_CHAT_ID -R you/repo --body "$YOUR_TG_USER_ID"

Pair your Telegram identity

Start the daemon. From your phone, DM the bot any message. The bot replies with a 6-character pairing code. In the running session, run: 

/telegram:access pair <code>
/telegram:access policy allowlist  # lock to allowlist after pairing

Now your Telegram ID is the only one that can drive the agent. Add partners by repeating the pairing.

The cheatsheet

Useful one-liners for living with this stack.

tmux attach -t claude-bot
Attach to the running daemon's terminal. Ctrl-b then d to detach.
launchctl list | grep amir.
Health check: are all 5 daemons running?
tail -f ~/.claude/logs/claude-bot.{out,err}.log
Live tail of the daemon's stdout + stderr.
tail -f ~/.claude/logs/prompts.log
Audit trail — every prompt that hit the daemon, with timestamps.
claude mcp list
MCP server health check.
ssh amir@<tailscale-ip>
Remote shell from your phone (after enabling Remote Login + signing into Tailscale on phone).
launchctl unload ~/Library/LaunchAgents/com.amir.claude-bot.plist && launchctl load same
Force-restart the daemon after config changes.
~/.claude/scripts/send-voice-reply.sh <chat_id> "<text>"
Make the agent send a voice note. Default voice Samantha (override with optional 3rd arg).

Things to say to it

Optimized prompts that exercise the full team. Voice or text — same syntax.

"audit reform for production readiness, run security + qa in parallel and tell me the verdict"
Triggers CEO → security-auditor + qa-engineer in parallel → consolidated verdict.
"plan how we'd add a refunds flow to reform stripe, then build it"
Architect first, then reform-engineer implements per the plan.
"take a screenshot of novalabs.ae mobile and tell me how the hero looks"
Playwright MCP, headed Chromium, mobile viewport. Returns screenshot + critique.
"open the gold-platform staging postgres and find hedges with negative pnl this week"
Postgres MCP query, results summarized in chat.
"draft 3 LinkedIn variants announcing reform 2.0, in [client]'s brand voice"
Marketing strategist; brand voice persisted in ~/.claude/agents/marketing-strategist.md.
"compose handover docs for reform and put a 5-min demo video in handover/"
Client-handover agent. Generates HANDOVER.md + ADMIN_GUIDE.md + Playwright-recorded demo.mp4.
"give me a voice reply with status of all 6 repos"
Returns a Telegram voice note (Samantha) + a text fallback for notifications.
